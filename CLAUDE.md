## Project

A single-file Python Telegram bot (`bot.py`) that recommends what to wear based on real-time local weather. Supports English and Ukrainian. Weather from Open-Meteo (free, no key). Outfit text generated by Pollinations.ai (free, no key).

## Commands

```bash
pip install -r requirements.txt   # install deps
python bot.py                     # run locally (requires .env with BOT_TOKEN)
```

## Architecture

All logic is in `bot.py`. Two in-memory dicts hold state between messages:

```python
user_locations: dict[int, tuple[float, float]]  # user_id → (lat, lon)
user_languages: dict[int, str]                  # user_id → "English" | "Ukrainian"
```

**User flow:**
1. `/start` → language inline buttons (`lang_en` / `lang_uk` callbacks)
2. Language chosen → location `ReplyKeyboardMarkup` (Telegram geo button)
3. Location received → stored; outfit inline button shown
4. Button / `/outfit` → `_send_outfit()` → weather + AI → reply

**Key functions:**

| Function | What it does |
|---|---|
| `fetch_weather(lat, lon)` | Open-Meteo `/v1/forecast?hourly=...` — returns `(temp, code, wind)` for the current UTC hour index |
| `fetch_outfit_suggestion(temp, code, wind, language)` | GET `https://text.pollinations.ai/{encoded_prompt}` — 10 s timeout, language-aware fallback on error |
| `_send_outfit(reply_fn, user_id, chat_id, context)` | Shared core used by both `/outfit` command and the inline button callback; sends typing action, calls both APIs, replies |
| `callback_language` | Sets `user_languages[user_id]`; routes to outfit button if location already known, else to location keyboard |

**Open-Meteo fields used:** `temperature_2m`, `weathercode`, `windspeed_10m` (hourly, UTC, forecast_days=1).

**Pollinations prompt shape:**  
`"You are a fashion assistant. Suggest a short outfit (2-3 items) for this weather: {temp}°C, {condition}, wind {wind}km/h. Reply in exactly one short paragraph, no lists, no line breaks. Casual tone. Reply in {language}."`

| Variable | Where |

| `BOT_TOKEN` | `.env` locally; Railway env vars in production |

## Deployment

- **Procfile:** `worker: python bot.py` (Railway picks this up automatically)
- No web server — bot uses long-polling (`app.run_polling()`)
- `python-telegram-bot` v21 is fully async; all handlers must be `async def`
